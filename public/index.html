Why Caracal

Designed around session flow, not generic distro bragging.

The goal is simple: remove setup drag, keep the base clean, and make Linux audio feel credible from the first boot through real production work.

Realtime first

Low-latency defaults without the post-install ritual.

Caracal ships with audio and realtime groups configured, rtprio 95, unlimited memlock, and a performance CPU governor so sessions start from a tuned baseline.

Studio core

Boot into a workstation with real DAWs and patch tools already there.

Ardour 9, Qtractor, and Carla are included out of the box, with REAPER, Renoise, and Bitwig available through simple post-install commands or the bundled installer.

Plugin depth

The stock stack actually covers writing, mixing, sound design, and guitar.

LSP, Dragonfly Reverb, Calf, BYOD, x42, Dexed, Odin2, AIDA-X, Neural Amp Modeler, and more are ready in LV2, VST3, and CLAP formats where available.

Hybrid workflows

Run Linux-native audio software and bridge in Windows VSTs when needed.

Wine TKG plus Yabridge are part of the story, so producers moving from Windows-heavy sessions are not boxed into a thin plugin ecosystem.

Workflow

From recording to synthesis to plugin-heavy mixes.

The supplied screenshots lean into the actual target audience: track builders, engineers, guitar-focused workflows, and anyone who needs both native Linux tools and a path for cross-platform plugins.

Stack

A deep stock setup with room to expand when the session asks for it.

Caracal is opinionated about the baseline but not restrictive about the ceiling. Keep the included stack, add commercial DAWs later, or bridge in Windows plugins when a project depends on them.

Bundled installer

Optional software feels curated instead of improvised.

The Caracal Software Installer presents DAWs, instruments, effects, and utilities in a guided catalog so the distro can stay lean while still covering serious production needs.

Install

Three clean moves to get from atomic desktop to audio workstation.

The install story is intentionally compact: switch the image, run the first-boot helper, then add anything extra through the software catalog.

01

Switch your atomic base

Move to the published bootc image from any compatible atomic desktop.

sudo bootc switch ghcr.io/caracal-dev/caracal:latest
02

Run first boot setup

Add the audio and realtime group setup, shell extras, and yabridge integration.

ujust first-run
03

Expand the studio as needed

Use the installer or the focused helpers when you want commercial DAWs or more instruments.

ujust software-installer

Verification

Published images are signed.

Verify the image signature before deployment if you want a tighter chain of trust.

cosign verify --key cosign.pub ghcr.io/caracal-dev/caracal:latest
